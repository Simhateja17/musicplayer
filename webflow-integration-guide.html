Component Communication & Data Flow

üîÑ How Components Work Together

The components use window.postMessage() for internal communication and pull all data directly from your Neon database:

  • Dashboard ‚Üí Library: "Browse Music Library" button scrolls to library section
  • Dashboard ‚Üí Player: "Open My Player" communicates with parent page for navigation
  • Library ‚Üí Dashboard: When songs are saved, dashboard refreshes automatically
  • All Components ‚Üí Database: Every song, playlist, and user data comes directly from Neon DB
‚úÖ Pure Database-Driven: No hardcoded data or fallback content. Everything pulls from your Neon database in real-time.

Database-Only Operation

üìã Important - No Fallback Data:
  • ‚ö†Ô∏è Components will show empty states if database is unavailable
  • ‚ö†Ô∏è No hardcoded songs or playlists - everything comes from Neon DB
  • ‚ö†Ô∏è Ensure your backend API is always accessible from Webflow domain
  • ‚ö†Ô∏è Navigation relies on parent page or component communication only

5.1 Required Database Setup:

-- Ensure these tables exist in your Neon database: CREATE TABLE songs ( id SERIAL PRIMARY KEY, title VARCHAR(255) NOT NULL, artist VARCHAR(255) NOT NULL, streaming_url TEXT NOT NULL, download_url TEXT, artwork_url TEXT, wav_url TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE user_playlists ( id SERIAL PRIMARY KEY, user_id VARCHAR(255) NOT NULL, song_id INTEGER REFERENCES songs(id), added_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, UNIQUE(user_id, song_id) );

5.2 Data Population:

Add songs to your database using the API endpoint or direct SQL:

-- Example: Add a song to your database INSERT INTO songs (title, artist, streaming_url, download_url, artwork_url, wav_url) VALUES ( 'Song Title', 'Artist Name', 'https://your-cdn.com/song.wav', 'https://your-cdn.com/song.mp3', 'https://your-cdn.com/artwork.jpg', 'https://your-cdn.com/song.wav' );
